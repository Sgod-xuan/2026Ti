# C07A 核心板接 NANO 转 UNO 板 README

本文档说明如何把 **WHEELTEC C07A 核心板（MSPM0G3507-48pin）** 通过飞线接到 **NANO 转 UNO 板**，再连接小车硬件。

## 1. 基本理解

实际硬件关系是：

```text
小车硬件
  |
NANO 转 UNO 板
  |
C07A 核心板
  |
MSPM0G3507 芯片
```

C07A 不是 CPU。C07A 是一块核心板，板上封装了 MSPM0G3507 芯片、USB-C、USB 转串口、稳压电路、晶振、按键和 H1/H2 排针。

你实际接线时接的是 **C07A 排针**，写代码时配置的是 **MSPM0G3507 的 PAx/PBx 引脚**。

可以这样记：

```text
接线看 C07A 排针丝印
写代码看 PAx/PBx
```

例如：

```text
C07A 排针 PA27 -> NANO 转 UNO 板 A0
代码里配置 PA27
```

## 2. C07A 排针和 G3507 引脚的关系

C07A 排针旁边标的 `PA27`、`PB3`、`PA0` 等名字，就是 MSPM0G3507 的引脚信号名。

文档里的 `H1-12`、`H2-5` 是排针位置编号，用来防止数错位置。

| 实物接线名 | MCU 信号名 | 说明 |
| --- | --- | --- |
| C07A 排针 `PA27` | `PA27` | 同一个信号，接线和代码都叫 PA27 |
| C07A 排针 `PB3` | `PB3` | 同一个信号，接线和代码都叫 PB3 |
| C07A 排针 `PA0` | `PA0` | 可配置为 I2C0 SDA |

## 3. 推荐接线总表

按当前项目的小车逻辑接线，推荐这样连接：

| NANO 转 UNO 板 Nano 插座 | C07A 排针丝印 | C07A 位置 | 用途 |
| --- | --- | --- | --- |
| `A0` | `PA27` | H1-12 | 巡线 S1，左外侧 |
| `A1` | `PA26` | H1-4 | 巡线 S2，左内侧 |
| `A2` | `PA25` | H1-3 | 巡线 S3，右内侧 |
| `A3` | `PA24` | H1-13 | 巡线 S4，右外侧 |
| `A4` | `PA0` | H1-6 | MPU6050 SDA |
| `A5` | `PA1` | H1-5 | MPU6050 SCL |
| `D9` | `PA7` | H2-3 | 左电机 AIN1/IN1，PWM |
| `D6` | `PB2` | H2-4 | 左电机 AIN2/IN2，PWM 或 GPIO |
| `D10` | `PB3` | H2-5 | 右电机 AIN1/IN1，PWM |
| `D5` | `PA17` | H2-6 | 右电机 AIN2/IN2，PWM 或 GPIO |
| `D2` | `PA12` | H1-9 | 左编码器 B 相，中断输入 |
| `D3` | `PA8` | H1-10 | 右编码器 B 相，中断输入 |
| `D7` | `PA9` | H1-11 | 左编码器 A 相 |
| `D8` | `PA13` | H2-9 | 右编码器 A 相 |
| `D4` | `PB17` | H1-7 | 备用外接启动按钮；当前代码改用 C07A 板载 SW1 |
| `D12` | `PA22` | H2-12 | HC-SR04 Trig，可选 |
| `D13` | `PA15` | H2-13 | HC-SR04 Echo，可选，建议分压 |
| `3V3` | `3V3` | H1-15 或 H2-1 | 3.3V 参考电源 |
| `+5V` | `5V` | H1-1 | 可选，只给 5V 电源轨供电 |
| `GND` | `GND` | H1-2/H1-14/H2-2 | 共地 |

## 4. Nano 插座脚位方向

NANO 转 UNO 板上原本插 Arduino Nano 的 30 针插座通常是：

```text
D1/TX  (1)   (30) VIN
D0/RX  (2)   (29) GND
RESET  (3)   (28) RESET
GND    (4)   (27) +5V
D2     (5)   (26) A0
D3     (6)   (25) A1
D4     (7)   (24) A2
D5     (8)   (23) A3
D6     (9)   (22) A4
D7    (10)   (21) A5
D8    (11)   (20) A6
D9    (12)   (19) A7
D10   (13)   (18) AREF
D11   (14)   (17) 3V3
D12   (15)   (16) D13
```

接线时优先看转接板上的 Nano 丝印和方向标记，不要只凭左右猜。

## 5. 串口 D0/D1 注意

当前不建议使用 NANO 转 UNO 板的 `D0/D1` 接 HC-05。

原因是 C07A 原理图中：

- `PA10` 已接板载 CH9102F USB 转串口
- `PA11` 已接板载 CH9102F USB 转串口

也就是说，G3507 的默认 UART0 已经用于 C07A 的 USB 下载/调试串口，并没有直接作为 H1/H2 普通排针给你接 Nano `D0/D1`。

如果后续必须接 HC-05，需要重新选择另一组 UART 引脚，并同步修改电机或其他引脚分配。

## 6. 按钮连接

当前项目原始逻辑里，启动按钮接在 Arduino 逻辑引脚 `D4`：

```text
Start button -> D4, INPUT_PULLUP, pressed is LOW
```

在 C07A 接线方案中，`D4` 映射为：

```text
Nano D4 -> C07A PB17 -> H1-7
```

但当前代码已经改为使用 C07A 板载 `SW1` 作为启动按钮：

```text
C07A SW1 -> PB8
按下电平：LOW
```

所以你现在不需要再接 Nano `D4` 的外部启动按钮。

如果以后要改回小车外接启动按钮，应这样接：

```text
按钮一端 -> C07A PB17 / H1-7
按钮另一端 -> GND
```

代码里再把启动输入从 `PB8` 改回 `PB17`，并配置输入上拉，按下时读到 LOW。

### 可以直接用 C07A 板载按钮吗

可以，但需要改代码映射。

从 C07A 原理图看：

- `SW1` 接到 `PB8`，可作为普通用户按钮。
- `SW2` 接到 `NRST`，是复位按钮，不能当启动按钮用。
- `SW3` 接到 `PA18`，用于 BSL/启动相关功能，不建议当小车启动按钮用。

当前代码已经选择 C07A 上的板载用户按钮 `SW1`。不要再接 Nano `D4` 的外部按钮。

```text
启动按钮：PB8
按下电平：LOW
```

推荐选择：

| 方案 | 接线 | 代码里读哪个引脚 | 建议 |
| --- | --- | --- | --- |
| 使用 C07A 板载 SW1 | 不接外部按钮 | `PB8` | 当前代码已采用 |
| 使用外接启动按钮 | `PB17/H1-7 -> 按钮 -> GND` | `PB17` | 需要再改代码 |
| 使用 C07A SW2 | 不建议 | `NRST` | 这是复位键，不能当启动键 |
| 使用 C07A SW3 | 不建议 | `PA18` | BSL/启动相关，避免占用 |

## 7. 供电规则

推荐先这样接：

```text
C07A 3V3 -> Nano 插座 3V3
C07A GND -> Nano 插座 GND
```

如果 NANO 转 UNO 板或小车外设确实需要 UNO 侧 5V 电源轨：

```text
C07A H1-1 5V -> Nano 插座 +5V
```

注意：

- 不要把 Nano `VIN` 接到 C07A，除非你确认转接板电源路径。
- 电机电池正极只接电机驱动 `VM`/`Motor VIN`。
- 电机电池正极不要接 C07A 的 `3V3` 或 `5V`。
- C07A、NANO 转 UNO 板、小车传感器、电机驱动、电机电池负极必须共地。

## 8. 电平规则

MSPM0G3507 是 3.3V 逻辑。TI 数据手册写的是 `Two 5V-tolerant open drain IOs`，不是所有 IO 都能直接接 5V。

因此：

- 输入 C07A 的信号默认按 3.3V 处理。
- LF04、编码器、HC-SR04 Echo 如果输出 5V，需要分压或电平转换。
- MPU6050 建议 3.3V 供电，SDA/SCL 使用 3.3V 上拉。
- 电机驱动输入必须能识别 3.3V 高电平。

## 9. 代码映射

后续实现 `platform_hw_*` 时，代码应按下列 G3507 信号配置：

| 功能 | G3507 信号 |
| --- | --- |
| 巡线 S1/S2/S3/S4 | `PA27 / PA26 / PA25 / PA24` |
| MPU6050 I2C | `PA0 SDA / PA1 SCL` |
| 左电机 | `PA7 / PB2` |
| 右电机 | `PB3 / PA17` |
| 编码器 B 相中断 | `PA12 / PA8` |
| 编码器 A 相 | `PA9 / PA13` |
| 启动按钮 | C07A 板载 SW1：`PB8` |
| 超声波可选 | `PA22 / PA15` |

当前 `src/platform/platform_mspm0.c` 中的 `platform_hw_*` 还是 weak 空实现，需要按上表补 GPIO、PWM、I2C 和中断配置。

当前启动按钮相关代码位置：

- `include/platform/c07a_pinmap.h`：定义 `C07A_START_BUTTON_SIGNAL_NAME` 为 `PB8`。
- `src/platform/platform_mspm0.c`：`platform_hw_start_button_pressed()` 按 SW1 低电平有效处理。
- 真正接入硬件时，实现 `platform_hw_c07a_sw1_pin_is_high()`，让它返回 PB8 的实际输入电平。

## 10. 上电前检查

- `3V3` 没有和外部 `5V` 短接。
- 所有 GND 已共地。
- 电机电源没有接到 C07A 的 `3V3` 或 `5V`。
- 进入 C07A 的高电平不超过 3.3V，除非明确确认该引脚可承受 5V。
- 先不接电机，只接传感器、按钮和 I2C 测试。
- 软件读数确认正确后，再接电机驱动并低 PWM 测试方向。
