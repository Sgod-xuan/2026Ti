# MSPM0G3507 + UNO 转接板小车接线指南

本文档按当前项目的逻辑引脚定义，说明如何把 MSPM0G3507 通过 UNO 转接板连接到小车硬件。

## 适用前提

- 主控为 TI MSPM0G3507，通常是 LP-MSPM0G3507 LaunchPad 或兼容板。
- UNO 转接板已经把 MSPM0G3507 的引脚引出成 Arduino UNO 风格丝印，例如 `D0`、`D1`、`A0`。
- 本项目源码当前使用的是 Arduino 风格逻辑引脚名，真实 MSPM0G3507 物理管脚和 pin mux 还没有在代码中实现。
- 接线时优先看 UNO 转接板上的丝印；如果转接板没有 UNO 丝印，必须查该转接板原理图再对照本文的逻辑引脚。

## 推荐接线步骤

1. 先只接电源地线。
   - MSPM0G3507 GND、UNO 转接板 GND、电机驱动 GND、传感器 GND、电机电池负极全部连在一起。
   - 电机电池正极只接电机驱动 `VM`/`Motor VIN`，不要接到 MSPM0G3507 的 5V 或 3V3。

2. 接巡线传感器。
   - `S1 -> A0`
   - `S2 -> A1`
   - `S3 -> A2`
   - `S4 -> A3`
   - `VCC -> 3V3`
   - `GND -> GND`

3. 接电机驱动控制线。
   - 左电机方向/驱动输入：`D9`、`D6`
   - 右电机方向/驱动输入：`D10`、`D5`
   - 如果左右方向反了，优先交换对应电机的两根输出线，或在代码里调整电机方向符号。

4. 接编码器。
   - 左编码器：`A -> D7`，`B -> D2`
   - 右编码器：`A -> D8`，`B -> D3`
   - 如果里程方向反了，交换 A/B 相或在代码里调整计数方向。

5. 接启动按钮。
   - 按钮一端接 `D4`
   - 另一端接 `GND`
   - 代码设计为内部上拉，按下时读到 LOW。

6. 接 MPU6050。
   - `SDA -> A4`
   - `SCL -> A5`
   - `VCC -> 3V3`
   - `GND -> GND`

7. 最后再接可选调试模块。
   - HC-05：`TXD -> D0`，`RXD -> D1`
   - HC-SR04：`Trig -> D12`，`Echo -> D13`，Echo 必须降压到 3.3V。

## 电平和供电注意事项

- MSPM0G3507 工作电源范围是 1.62V 到 3.6V，LaunchPad 目标板常用 3.3V 逻辑。
- TI 数据手册写的是 `Two 5V-tolerant open drain IOs`，不是所有 IO 都能接 5V。
- 接线时不要默认 UNO 转接板上的每个引脚、每种外设复用状态都能直接承受 5V。
- 未确认具体 MSPM0G3507 物理引脚和转接板原理图前，所有输入 MSPM0G3507 的模块信号建议按 0V 到 3.3V 处理：
  - LF04 如果 5V 供电且输出高电平为 5V，建议电平转换或电阻分压。
  - 编码器如果输出 5V，建议电平转换或电阻分压。
  - HC-SR04 的 Echo 通常为 5V，建议分压。
  - HC-05 的 TXD 通常为 3.3V TTL，可直接进主控 RX；HC-05 的 RXD 端建议串分压。
- 电机不能直接接 MCU 引脚，必须通过电机驱动板。
- 电机电源和主控电源可以分开，但 GND 必须共地。

## 与当前代码的关系

项目接线定义来自：

- `docs/hardware.md:11-41`

当前代码还没有真正配置 MSPM0G3507 的 GPIO/PWM/I2C/UART 外设：

- `src/platform/platform_mspm0.c:219-280` 中的 `platform_hw_*` 函数还是 weak 空实现。
- 后续需要在 `platform_hw_init()` 中把上表逻辑引脚映射到真实 MSPM0G3507 管脚，并配置 GPIO、PWM、编码器中断、I2C 和 UART。

## 上电前检查清单

- 电机电源没有接到 MSPM0G3507 的 3V3 或 5V 输出。
- 所有模块 GND 已共地。
- 未确认 5V tolerant 的输入脚，高电平不超过 3.3V。
- 左右电机没有直接接 MCU 引脚。
- MPU6050 的 SDA/SCL 已接到 `A4/A5`。
- 启动按钮按下时 `D4` 会被拉到 GND。
- HC-SR04 Echo 已经分压后再进 `D13`。
